# 多语言（@packages/i18n）

本仓库微前端的多语言方案：**i18next + react-i18next**，由 workspace 包 `@packages/i18n` 统一封装。主语言为 **简体中文（`zh-CN`）**，当前内置 **英文（`en`）**。

## 两层含义：基础设施 vs 页面文案

| 层级 | 含义 | 当前状态 |
| --- | --- | --- |
| **基础设施** | `package.json` 依赖、`I18nProvider`、`AntdLocaleProvider`、命名空间在 `resources.ts` 注册 | **全部完成**（main + 6 个子应用） |
| **页面文案** | 业务 UI 使用 `useT('<ns>')`，词条写在 `locales/*/<ns>.ts` | **仅 login 完成**；agent / blog / skill / resume / utils / main 菜单等仍为中文硬编码 |

切换语言后，**即使页面文案未迁移**，各应用内 **Ant Design 内置文案**（日期、分页、表单校验等）也会随 `private_locale` 变化。

## 设计原则

| 项 | 约定 |
| --- | --- |
| 语言偏好存储 | 全站唯一键 **`localStorage['private_locale']`**（与 `private-main-theme` 命名风格一致） |
| 命名空间 | 按业务拆分：`common`（语言切换等）、`login`、`agent`、`blog` … |
| 子应用接入 | 各自声明 `I18nProvider` + 所需 `namespaces`；**禁止**再使用 `login:locale` 等应用级键 |
| Ant Design | 通过 `AntdLocaleProvider` / `getAntdLocale` 与文案语言对齐 |
| 多实例同步 | 主应用与子应用各自 `I18nProvider` 实例，通过 `storage` 事件 + `private_locale-change` 同页同步 |

## 已接入应用

| 应用 | 基础设施 | 页面文案 | 语言切换 UI | 随语言 SEO |
| --- | --- | --- | --- | --- |
| **main** | ✅ | ⏳ 菜单、控制台等硬编码 | 顶栏 `SiteLocaleSwitcher`（`/login` 时隐藏顶栏） | ❌ |
| **login** | ✅ `namespaces={['login']}` | ✅ `useT('login')` | 登录壳 `AuthShell` 内切换器 | ✅ `LoginSeoSync` |
| **agent** | ✅ | ⏳ | 依赖 main 顶栏 | ❌ |
| **blog** | ✅ | ⏳ | 同上 | ❌ |
| **skill** | ✅ | ⏳ | 同上 | ❌ |
| **resume** | ✅ | ⏳ | 同上 | ❌ |
| **utils** | ✅ | ⏳ | 同上 | ❌ |

子应用根组件统一模式（`apps/<name>/src/App.tsx`）：

```tsx
<I18nProvider namespaces={['<app>']}>
  <AntdLocaleProvider theme={/* 可选 */}>
    <RouterProvider router={router} />
  </AntdLocaleProvider>
</I18nProvider>
```

### main（主应用）

- `apps/main/src/App.tsx`：`<I18nProvider>`（默认仅加载 `common`）。
- `apps/main/src/theme/ThemeRoot.tsx`：`<AntdLocaleProvider theme={...}>`（含亮/暗 algorithm）。
- `apps/main/src/layouts/index.tsx`：`<SiteLocaleSwitcher />`。

主应用菜单国际化可在 `common` 或新增 `main` 命名空间后，为 `I18nProvider` 增加 `namespaces={['main']}` 并逐步替换硬编码。

### login（子应用）

- 页面：`useT('login')` + `t('signin.welcome')` 等（`singin`、`register`）。
- 语言切换：登录页右上角 `SiteLocaleSwitcher`（与 main 共用 `private_locale`）。
- SEO：`LoginSeoSync` 从 `login` 命名空间的 `seo.*` 读取并调用 `applyDocumentSeo`。

> 进入 `/login` 时主应用顶栏隐藏，仅在 login 壳层显示切换器；其它路由用 main 顶栏切换，效果一致。

### 其它子应用（agent / blog / skill / resume / utils）

- 已挂载 i18n，Ant Design 组件随全站语言切换。
- 业务 UI、表格 `locale.emptyText`、部分页面内中文仍为硬编码。
- 改造步骤见下文「迁移既有页面文案」。

## 快速接入新子应用

### 方式 A：脚手架（推荐）

```bash
pnpm micro-app:create
```

生成的 `App.tsx` 已包含 `I18nProvider` + `AntdLocaleProvider` 与 `@packages/i18n` 依赖。仍需 **手动** 增加文案文件并在 `resources.ts` 注册（见方式 B 第 1 步）。

### 方式 B：手动（或脚手架之后补文案）

#### 1. 增加文案

`packages/i18n/src/locales/zh-CN/demo.ts`、`en/demo.ts`，并在 `resources.ts` 注册：

```ts
import { demoZhCN } from './locales/zh-CN/demo.ts';
import { demoEn } from './locales/en/demo.ts';

export const i18nResources = {
  'zh-CN': { /* ...existing */, demo: demoZhCN },
  en: { /* ...existing */, demo: demoEn },
} as const;
```

#### 2. 根组件

`apps/demo/src/App.tsx`：

```tsx
import { AntdLocaleProvider, I18nProvider } from '@packages/i18n';

export default function App() {
  return (
    <I18nProvider namespaces={['demo']}>
      <AntdLocaleProvider>
        {/* 路由 */}
      </AntdLocaleProvider>
    </I18nProvider>
  );
}
```

#### 3. 页面中使用

```tsx
import { useT } from '@packages/i18n';

function DemoPage() {
  const { t } = useT('demo');
  return <h1>{t('title')}</h1>;
}
```

#### 4.（可选）独立全屏壳层的语言切换

```tsx
import { SiteLocaleSwitcher } from '@packages/i18n';

<SiteLocaleSwitcher className="your-locale-select-class" />
```

## 迁移既有页面文案（清单）

对某个子应用（以 `utils` 为例）：

1. 在 `packages/i18n/src/locales/zh-CN/utils.ts`、`en/utils.ts` 按页面模块组织 key（保持 zh/en 结构一致）。
2. 将页面中的中文字符串改为 `const { t } = useT('utils')`；Ant Design 的 `locale={{ emptyText: '...' }}` 等一并迁入词条。
3. （可选）若需随语言更新 SEO：在命名空间增加 `seo` 字段，参考 `apps/login/src/components/LoginSeoSync.tsx`。
4. 本地切换 `zh-CN` / `en` 走查；注意 qiankun 嵌入与独立打开两种入口。

仓库内 **`useT(` 仅出现在 `apps/login`**，可作为迁移参考。

## 对外 API

| 导出 | 说明 |
| --- | --- |
| `I18nProvider` | 创建 i18n 实例；`namespaces` 为业务命名空间（自动包含 `common`） |
| `AntdLocaleProvider` | 根据 `useLocale()` 设置 Ant Design `ConfigProvider.locale` |
| `SiteLocaleSwitcher` | 语言下拉（`zh-CN` / `en`） |
| `useT(ns)` | 返回 `useTranslation(ns)`，**必须**传入命名空间 |
| `useLocale()` | `{ locale, setLocale }`，写入 `private_locale` 并广播 |
| `getAntdLocale(locale)` | `antd/locale/zh_CN` 或 `en_US` |
| `SITE_LOCALE_STORAGE_KEY` | `'private_locale'` |
| `DEFAULT_LOCALE` / `SUPPORTED_LOCALES` | 类型与默认值 |
| `I18nNamespace` | 已注册命名空间联合类型 |

## 遗留数据迁移

首次读取时若存在以下键，会自动迁移到 `private_locale` 并删除旧键：

- `private:locale`
- `login:locale`
- `app:locale`

## 与 SEO 包协作

- **login**：`LoginSeoSync` 在 `locale` 变化时调用 `applyDocumentSeo`，文案来自 `login` 命名空间的 `seo.*`。
- **其它应用**：`main.tsx` 仍使用 `appSeoPresets` 中文预设；`applyDocumentSeo` 当前固定 `document.documentElement.lang = 'zh-CN'`（见 [seo.md](./seo.md)）。若子应用要做多语言 SEO，需同时扩展 SEO 包或在 `applyDocumentSeo` 调用处传入与 `locale` 一致的 `lang` / `og:locale`（尚未实现）。

## 常见问题

**Q：main 与子应用各包一层 `I18nProvider` 会冲突吗？**

不会。各自独立实例，共享 `localStorage` 与 `private_locale-change`；切换后已挂载实例会 `changeLanguage`。

**Q：`useT()` 不写命名空间可以吗？**

不可以。必须显式传入，例如 `useT('login')`。

**Q：子应用已挂 Provider，为什么页面还是中文？**

基础设施只保证 Ant Design 与语言偏好同步；业务字符串需按「迁移清单」写入命名空间并改用 `useT`。
