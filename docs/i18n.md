# 多语言（@packages/i18n）

本仓库微前端的多语言方案：**i18next + react-i18next**，由 workspace 包 `@packages/i18n` 统一封装。主语言为 **简体中文（`zh-CN`）**，当前内置 **英文（`en`）**。

## 设计原则

| 项 | 约定 |
| --- | --- |
| 语言偏好存储 | 全站唯一键 **`localStorage['private_locale']`**（与 `private-main-theme` 命名风格一致） |
| 命名空间 | 按业务拆分：`common`（语言切换等）、`login`（登录注册页）等 |
| 子应用接入 | 各自声明 `I18nProvider` + 所需 `namespaces`；**禁止**再使用 `login:locale` 等应用级键 |
| Ant Design | 通过 `AntdLocaleProvider` / `getAntdLocale` 与文案语言对齐 |
| 多实例同步 | 主应用与子应用各自 `I18nProvider` 实例，通过 `storage` 事件 + `private_locale-change` 同页同步 |

## 已接入应用

### main（主应用）

- 根组件：`App.tsx` 包裹 `<I18nProvider>`（默认仅加载 `common`）。
- 主题：`ThemeRoot` 内使用 `<AntdLocaleProvider theme={...}>`。
- 顶栏：`layouts/index.tsx` 中 `<SiteLocaleSwitcher className="main-header-locale-select" />`，位于主题切换左侧。

### login（子应用）

- 根组件：`<I18nProvider namespaces={['login']}>` + `<AntdLocaleProvider theme={loginTheme}>`。
- 页面文案：`useT('login')` + `t('signin.welcome')` 等。
- 语言切换：登录页右上角 `<SiteLocaleSwitcher className="login-locale-switcher" />`（与 main 读写同一 `private_locale`）。
- SEO：`LoginSeoSync` 根据 `login` 命名空间下 `seo.*` 与当前语言更新 `document` 元数据。

> 进入 `/login` 时主应用顶栏隐藏，仅在 login 壳层显示切换器；在其它路由用 main 顶栏切换，效果一致。

## 快速接入新子应用

以 hypothetical `demo` 子应用为例：

### 1. 增加文案

`packages/i18n/src/locales/zh-CN/demo.ts`、`en/demo.ts`，并在 `resources.ts` 注册：

```ts
import { demoZhCN } from './locales/zh-CN/demo.ts';
import { demoEn } from './locales/en/demo.ts';

export const i18nResources = {
  'zh-CN': { common: commonZhCN, login: loginZhCN, demo: demoZhCN },
  en: { common: commonEn, login: loginEn, demo: demoEn },
} as const;
```

### 2. 依赖与根组件

`apps/demo/package.json`：

```json
"dependencies": {
  "@packages/i18n": "workspace:*"
}
```

`apps/demo/src/App.tsx`：

```tsx
import { AntdLocaleProvider, I18nProvider } from '@packages/i18n';

export default function App() {
  return (
    <I18nProvider namespaces={['demo']}>
      <AntdLocaleProvider>
        {/* 路由与子页面 */}
      </AntdLocaleProvider>
    </I18nProvider>
  );
}
```

### 3. 页面中使用

```tsx
import { useT } from '@packages/i18n';

function DemoPage() {
  const { t } = useT('demo');
  return <h1>{t('title')}</h1>;
}
```

### 4.（可选）顶栏切换

若子应用有独立全屏壳层、不显示 main 顶栏，可放置：

```tsx
import { SiteLocaleSwitcher } from '@packages/i18n';

<SiteLocaleSwitcher className="your-locale-select-class" />
```

## 对外 API

| 导出 | 说明 |
| --- | --- |
| `I18nProvider` | 创建子应用 i18n 实例；`namespaces` 为业务命名空间列表（会自动包含 `common`） |
| `AntdLocaleProvider` | 子组件内根据 `useLocale()` 设置 Ant Design `ConfigProvider.locale` |
| `SiteLocaleSwitcher` | 语言下拉（`zh-CN` / `en`） |
| `useT(ns)` | 返回 `useTranslation(ns)`，**必须**传入命名空间 |
| `useLocale()` | `{ locale, setLocale }`，写入 `private_locale` 并广播同步 |
| `getAntdLocale(locale)` | 映射为 `antd/locale/zh_CN` 或 `en_US` |
| `SITE_LOCALE_STORAGE_KEY` | 常量 `'private_locale'` |
| `DEFAULT_LOCALE` / `SUPPORTED_LOCALES` | 类型与默认值 |
| `I18nNamespace` | 已注册命名空间联合类型 |

## 遗留数据迁移

首次读取时若存在以下键，会自动迁移到 `private_locale` 并删除旧键：

- `private:locale`（旧全站键）
- `login:locale`
- `app:locale`

## 与 SEO 包协作

子应用若需随语言更新 `document` title/description，可参考 `apps/login/src/components/LoginSeoSync.tsx`：在对应命名空间增加 `seo` 字段，在 `useEffect` 中调用 `@packages/seo` 的 `applyDocumentSeo`。

## 常见问题

**Q：main 与子应用各包一层 `I18nProvider` 会冲突吗？**

不会。各自独立实例，共享同一份 `localStorage` 与 `private_locale-change` 事件；切换后其它已挂载实例会 `changeLanguage`。

**Q：`useT()` 不写命名空间可以吗？**

不可以。必须显式传入，例如 `useT('login')`，避免误用默认命名空间。

**Q：菜单、首页控制台文案何时国际化？**

尚未接入；可在 `common` 或新增 `main` 命名空间后，仅在 main 的 `I18nProvider` 上增加 `namespaces={['main']}` 并逐步替换硬编码中文。
